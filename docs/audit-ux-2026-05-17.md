# Audit UX dashboard Tik — 2026-05-17

**Contexte** : audit demandé par l'utilisatrice 2 jours après la refonte
Home tabs Marché/Calibration/Système (Paquet 24) et 7 jours avant le
démarrage du trading manuel J+14 (2026-05-14 décalé à 2026-05-24 selon
les engagements Garde-fou 2-bis).

**Méthodologie** : lecture intégrale de 11 fichiers (Home `index.tsx`
430 lignes, Watchlist, Signals, Config, 7 composants
`components/dashboard/*`, badge anti fake-news). Croisée avec le profil
trader débutante J+14 et le Garde-fou 2-bis (sizing 1 %, veracity ≥
0.90, discipline macro ±4 h). Engagements méthodologiques actifs
(CLAUDE.md section 13bis) respectés : pas d'invention, citation fichier
quand possible, format pour/contre/verdict pour chaque proposition.

---

## 🔴 Frictions critiques (à fixer avant trading manuel)

### F1. Pas de "checklist avant trade" sur Marché

**Constat** : ouvrir Tik en mobilité → aucune indication immédiate de
*"puis-je trader maintenant ?"*. Les 5 critères de Garde-fou 2-bis
sont éparpillés dans 4 cartes différentes (veracity dans VeracityGauge
+ MiniSparkline, événement macro proche dans MacroEventsCard, sizing
1 % nulle part exposé, flag AFN dans AntiFakeNewsBadge enfoui dans
Signals, marché GOLD ouvert nulle part exposé).

**Impact** : la trader doit scroller + croiser mentalement. À 22 h le
soir, fatiguée, **risque réel d'oubli d'un critère**.

| Option | Pour | Contre |
|---|---|---|
| A — Bande "feu vert / orange / rouge" en haut de Marché | Compact, immédiat, pédagogique | Calcul dépend de plusieurs hooks (composite logic) |
| B — Modal pré-trade déclenché depuis Watchlist | Évite la pollution Home | Force action explicite |
| C — Liste à cocher manuelle dans détail signal | Implication consciente trader | Manuel = oubli possible |

**Verdict** : **Option A** dès maintenant, **Option C** en backup post-J+30.
Effort ~1-2 h.

### F2. Densité Watchlist : 6 infos sur 1-2 lignes

**Constat** (`watchlist.tsx`) : chaque entry empile `entity · direction
· horizon · veracity% · confidence% · outcome` côte-à-côte. Sur iPhone
le flexwrap casse l'alignement.

**Verdict** : 2 lignes lisibles → ligne 1 `entity · direction ·
horizon`, ligne 2 `outcome · veracity%`. Cacher confidence% (déjà dans
le détail). Effort ~30 min.

### F3. Vocabulaire technique non expliqué (impact transversal)

**Termes vus, jamais définis nulle part dans l'app** : `conv` / `verac`
/ `horizon flash/swing/macro` / `circuit_breaker_status` / `seuil
0,70` / `dispersion` / `outcome` / `track record`.

| Option | Pour | Contre |
|---|---|---|
| A — Icône `?` tapable à côté de chaque terme → tooltip 2 lignes | Granulaire, opt-in | 20+ tooltips à écrire |
| B — Section "Glossaire" repliée dans Config | Centralisé, peu d'effort | Nécessite navigation hors contexte |
| C — Long-press sur un libellé → définition | Élégant, pas d'icônes parasites | Pas évident pour utilisatrice débutante |

**Verdict** : **A + B combinés**. A pour les 6 termes les plus
critiques (verac, conv, AFN, track record, horizon, seuil), B pour le
glossaire complet. Effort ~2 h.

### F4. `APP_VERSION = '0.5.7'` hardcodé dans `index.tsx:30`

**Constat** : `package.json` vient d'être bumpée à 0.5.9, mais Home
affiche encore 0.5.7. Bug de désynchronisation à chaque bump.

**Verdict** : importer depuis `package.json` (ou exposer via
`Constants.expoConfig`). Effort 5 min.

## 🟡 Frictions moyennes

### F5. Sélecteurs HitRate non synchronisés avec TopHeadlines

**Constat** : `index.tsx:107` → `headlinesEntity` indépendant ;
`index.tsx:113` → `hitRateEntity` indépendant. Changer "BTC" → "GOLD"
dans une carte ne propage pas aux autres.

| Option | Pour | Contre |
|---|---|---|
| A — Toolbar globale "BTC / GOLD" en haut du tab Marché qui pilote tout | UX cohérente, 1 seul tap | Refacto state + props drilling |
| B — Garder indépendants (statu quo) | Flexibilité, parfois on veut voir GOLD news + BTC hit rate | Cognitive load |

**Verdict** : **A** est plus simple pour débutante. À valider avec
l'utilisatrice : préfère-t-elle synchronisé ou indépendant ?

### F6. Featured event macro **ressemble** à un bouton mais ne l'est pas

**Constat** : `macro-events-card.tsx` — la grosse carte "FOMC dans 5j"
a une bordure colorée + countdown → affordance forte de cliquabilité.
Mais `Pressable` absent. Tap → rien.

**Verdict** : la rendre tappable → push `/macro` ou ajouter chevron `›`
indiquant "voir détail". Effort ~10 min.

### F7. Mini-sparkline veracity : légende cryptique

**Constat** : `mini-sparkline.tsx` affiche "tirets : seuils 0,70 et
0,85" en opacity 0.5, 10 px. Une trader débutante ne sait pas pourquoi
0.70 et 0.85, ni que **0.90 est SON seuil personnel** (Garde-fou
2-bis).

**Verdict** : ajouter une ligne 0.90 colorée + label "ton seuil".
Effort ~20 min.

### F8. `Colors.light.tint` hardcodé en plusieurs endroits

**Constat** : grep rapide sur `app/(tabs)/` → 4 occurrences de
`Colors.light.tint` au lieu de `palette.tint` ou
`Colors[colorScheme].tint`. Les pills actives en dark mode utilisent
la couleur light → mauvais contraste.

**Verdict** : remplacer par `palette.tint`. Effort ~15 min.

## 🟢 Frictions mineures

### F9. Logo About "?" dans le header parallax

**Constat** : `about.tsx:16` headerImage = literal `?` en 200 px.
Visuel pauvre, peu pro. Moins critique maintenant que l'onglet est
caché (Paquet 25 livré 2026-05-17), mais reste visible si on accède à
`/about` directement.

### F10. Bouton remove Watchlist = `×` minuscule sans confirmation

**Constat** : `watchlist.tsx` — position absolue top-right, pas de
modal. Tap accidentel = perte de la suivi sur 5j-30j.

**Verdict** : ajouter `Alert.alert("Retirer ce signal ?", ...)` natif
iOS. Effort 5 min.

### F11. Pas de pull-to-refresh sur Marché / Calibration

**Constat** : pattern iOS standard absent. Trader doit attendre les
polls automatiques.

**Verdict** : `RefreshControl` sur ScrollView. Effort ~20 min par tab.

## Limites de l'audit (4 — engagement 13bis)

1. **Pas d'observation runtime des vraies frictions de l'utilisatrice**
   : audit basé sur lecture de code, pas sur observation directe.
   Les choix de priorité peuvent être à côté de ce qui bloque
   réellement.
2. **Pas testé sur iPhone SE** : les contraintes "petit écran" sont
   inférées du code (flexwrap, padding), pas vérifiées.
3. **Pas d'A/B testing** : toutes les recommandations sont des
   intuitions UX argumentées, pas des résultats mesurés.
4. **Le tab system Marché/Calibration/Système (Paquet 24, 2 jours)
   n'est pas remis en cause** dans cet audit — suppose que
   l'utilisatrice valide ce niveau-là. Si elle veut le reconsidérer,
   à dire explicitement.
